# 화면 및 UX 기획 (Screen Specifications)

> 본 문서는 PRD 요구사항을 세분화하여, smart-portfolio 서비스에 필요한 **총 8개의 핵심 화면(사용자용 4개, 관리자용 4개)**을 도출하고 정리한 기획 문서입니다.

---

## 👨‍💻 방문자(사용자)용 화면

### 1. 메인 프로필 화면 (Home)
- **화면이름:** 메인 프로필 및 포트폴리오 요약
- **URL경로:** `/`
- **이 화면의 목적:** 개발자의 핵심 역량, 한 줄 소개, 기술 스택을 직관적으로 보여주는 랜딩 페이지.
- **관련 PRD 기능 번호:** 기능 2 (AI 대화형 포트폴리오 인터페이스)
- **기타 UI 설계 정보:** 화면 우측 하단에 떠 있는(Floating) 챗봇 버튼을 항시 노출. 깔끔하고 모던한 타이포그래피 활용.

### 2. 프로젝트 상세 화면 (Project Detail)
- **화면이름:** 프로젝트 상세 정보
- **URL경로:** `/project/:id` (예: `/project/smart-portfolio`)
- **이 화면의 목적:** 개발자가 진행한 특정 프로젝트의 아키텍처, 사용 기술, 기여도 등을 상세히 보여줌.
- **관련 PRD 기능 번호:** 기능 1 (포트폴리오 데이터 기반)
- **기타 UI 설계 정보:** 스크린샷 갤러리 및 GitHub/배포 링크 버튼 포함. 내용이 길어질 경우 스크롤에 맞춰 챗봇 버튼이 따라다니도록 설계.

### 3. 이력 및 경험 화면 (Resume)
- **화면이름:** 이력서 및 타임라인
- **URL경로:** `/resume`
- **이 화면의 목적:** 경력, 학력, 자격증 등 상세 이력을 타임라인 형태로 한눈에 보여줌.
- **관련 PRD 기능 번호:** 기능 1 (포트폴리오 데이터 기반)
- **기타 UI 설계 정보:** 모바일 화면에서 세로 타임라인으로 깨지지 않게 렌더링(반응형). PDF 다운로드 버튼 제공.

### 4. AI 챗봇 전용 화면 (AI Chatbot View)
- **화면이름:** AI 챗봇 풀스크린 대화창
- **URL경로:** `/chat`
- **이 화면의 목적:** 방문자가 포트폴리오 탐색 중 AI에게 집중적으로 질문을 던지고 답변을 받을 수 있는 전용 대화 화면 (특히 모바일 환경에 유용).
- **관련 PRD 기능 번호:** 기능 2 (AI 대화형 포트폴리오 인터페이스 및 예외 처리)
- **기타 UI 설계 정보:** 메신저 앱(예: 카카오톡, 슬랙)과 유사한 친숙한 UI. 추천 질문 칩(Chip) 제공 및 예외 처리 에러 메시지 스타일링.

## ⚙️ 포트폴리오 소유자(관리자)용 화면

### 5. 관리자 로그인 화면 (Admin Login)
- **화면이름:** 관리자 접근 인증
- **URL경로:** `/admin/login`
- **이 화면의 목적:** 포트폴리오 소유자가 관리자 대시보드에 접근하기 위한 보안 인증.
- **관련 PRD 기능 번호:** 기능 3 (접근 제어 로직)
- **기타 UI 설계 정보:** 중앙 정렬된 심플한 로그인 폼. 소셜 로그인(OAuth) 버튼 위주로 구성. 미인증 접근 시 에러 피드백.

### 6. 인사이트 대시보드 화면 (Dashboard Overview)
- **화면이름:** 방문자 통계 요약 대시보드
- **URL경로:** `/admin` (또는 `/admin/dashboard`)
- **이 화면의 목적:** 전체 방문자 수, 챗봇 사용 횟수, 가장 많이 물어본 키워드 통계를 한눈에 확인.
- **관련 PRD 기능 번호:** 기능 3 (방문자 인사이트 대시보드)
- **기타 UI 설계 정보:** 워드 클라우드나 Bar/Pie 차트 등 시각화 라이브러리 활용. 좌측 사이드바에 관리자 메뉴(대시보드, 로그, 설정) 배치.

### 7. 대화 로그 상세 화면 (Chat Logs)
- **화면이름:** 챗봇 대화 히스토리
- **URL경로:** `/admin/logs`
- **이 화면의 목적:** 방문자와 AI 챗봇이 나눈 실제 대화 내역을 조회하여 면접 예상 질문을 파악하고 답변 품질을 모니터링.
- **관련 PRD 기능 번호:** 기능 3 (방문자 인사이트 대시보드)
- **기타 UI 설계 정보:** 날짜별/키워드별 필터링 및 검색 기능 제공. 대화 내용을 테이블 또는 카드 리스트 형태로 페이징(Pagination) 처리.

### 8. 포트폴리오 지식 기반 설정 화면 (RAG Setup)
- **화면이름:** AI 학습 데이터 관리 및 동기화
- **URL경로:** `/admin/setup`
- **이 화면의 목적:** 개발자가 이력서(텍스트/마크다운)를 직접 수정하거나 GitHub 링크를 연동하여 AI 챗봇의 답변 기반 데이터를 업데이트.
- **관련 PRD 기능 번호:** 기능 1 (포트폴리오 지식 기반 자동 구축 시스템)
- **기타 UI 설계 정보:** 마크다운 에디터 탑재. "데이터 업데이트" 버튼 클릭 시 상태를 보여주는 스피너/프로그레스 바 필수. 성공/실패 토스트 알림 적용.

## 🎨 화면별 상세 UI 명세

지금까지 도출된 핵심 화면들에 대한 레이아웃, 섹션별 상세 정보 및 반응형 동작 명세입니다.

### 1. 메인 페이지 (/)

**레이아웃 구조**
- 상단 : 네비게이션바 (로고 왼쪽, 메뉴 오른쪽)
- 히어로 섹션 : 프로필 사진(원형) + 이름 + 한 줄 소개 + CTA 버튼
- 스킬 섹션 : 보유 스킬을 태그/뱃지 형태로 나열
- 경력 섹션 : 타임라인 형태의 경력 목록
- 하단 : 푸터 (연락처 정보, SNS 및 GitHub 링크)
- 공통(플로팅) : 우측 하단 AI 챗봇 호출 버튼 항시 노출

**각 섹션별 상세**
- 히어로 섹션의 CTA버튼 : “프로젝트 보러가기” -> `/project` 로 이동
- 프로필 사진 : 200*200px 원형, 가운데 정렬
- 스킬 태그 : 둥근 모서리 뱃지, 배경색 없이 외곽선(Outline)과 모던한 타이포그래피 활용
- 경력 타임라인 : 연도 / 회사명 / 역할 / 간략설명 순으로 구성
- AI 챗봇 버튼 : 스크롤을 내려도 화면 우측 하단에 고정(Fixed)되어 따라다니도록 설계

**반응형 동작**
- 모바일 : 1열 세로 배치, 네비게이션 메뉴는 햄버거 메뉴로 전환, AI 챗봇 버튼은 모바일 화면에 맞게 크기 및 여백 최적화
- 데스크톱 : 히어로 섹션 좌우 분할 (사진 왼쪽, 텍스트 오른쪽), 네비게이션 메뉴 가로로 전체 노출

---

### 2. 프로젝트 목록 페이지 (/project)

**레이아웃 구조**
- 상단 : 네비게이션바 (공통)
- 필터 섹션 : 카테고리별 필터 버튼 영역 (전체 / 웹 / 앱 / 디자인 등)
- 프로젝트 리스트 섹션 : 프로젝트 카드들이 나열되는 그리드(Grid) 영역
- 하단 : 푸터 (공통)
- 공통(플로팅) : 우측 하단 AI 챗봇 호출 버튼

**각 섹션별 상세**
- 프로젝트 카드 구성 : 썸네일 이미지 + 프로젝트 제목 + 카테고리 태그 + 간략 설명
- 호버(Hover) 효과 : 카드에 마우스 커서를 올리면 카드가 살짝 확대(Scale up)되며 부드러운 그림자(Drop shadow) 강조 효과 적용
- 클릭 시 동작 : 카드 클릭 시 해당 프로젝트의 상세 페이지(`/project/[id]`)로 이동
- 필터 버튼 : 선택된 카테고리에 하이라이트 효과(예: 색상 반전 또는 밑줄) 적용

**반응형 동작**
- 모바일 : 1열 그리드 배치 (카드가 화면 꽉 차게 세로로 나열)
- 태블릿 : 2열 그리드 배치
- 데스크톱 : 3열 그리드 배치

---

### 3. 프로젝트 상세 페이지 (/project/[id])

**레이아웃 구조**
- 상단 : 네비게이션바 (공통)
- 히어로 섹션 : 프로젝트 대표 역할을 하는 큰 이미지(또는 스크린샷 갤러리)
- 콘텐츠 섹션 : 프로젝트 상세 설명 + 사용 기술 스택 + 관련 외부 링크
- 하단 네비게이션 : 이전/다음 프로젝트로 이동할 수 있는 링크 버튼 영역
- 하단 : 푸터 (공통)
- 공통(플로팅) : 내용이 길어져도 스크롤에 맞춰 따라다니는 AI 챗봇 버튼

**각 섹션별 상세**
- 큰 이미지 : 화면 상단에 꽉 차거나, 내용 위쪽에 넓게 배치하여 시각적 임팩트 제공
- 상세 설명 : 프로젝트 개요, 주요 아키텍처, 기여한 부분 등을 단락별로 깔끔하게 구성
- 사용 기술 : 사용된 언어/프레임워크를 둥근 뱃지 형태로 모아서 배치
- 관련 링크 : GitHub 저장소, 라이브 데모(배포 사이트) 등으로 이동하는 눈에 띄는 버튼 제공
- 이전/다음 네비게이션 : 현재 프로젝트의 전/후 프로젝트 제목을 노출하여 클릭 시 바로 이동할 수 있도록 구성

**반응형 동작**
- 모바일 : 모든 콘텐츠 1열 세로 배치, 대표 이미지는 화면 너비 100%에 맞춤
- 데스크톱 : 가독성을 위해 텍스트 콘텐츠의 최대 너비(Max-width)를 제한하고 중앙 정렬. 텍스트와 스크린샷 갤러리를 좌우 분할하여 배치하는 것도 고려 가능

---

### 4. AI 챗봇 위젯 (공통 플로팅 컴포넌트)

**레이아웃 구조**
- 위치 : 모든 페이지의 오른쪽 하단에 고정된(Fixed) 플로팅 버튼
- 닫힌 상태 : 채팅 아이콘이 들어간 둥근 버튼
- 열린 상태 : 대화가 가능한 직사각형 채팅 패널
  - 상단 헤더 : “포트폴리오 AI 어시스턴트” 제목 + 창 닫기(X) 버튼
  - 중앙 대화 영역 : 사용자와 AI의 메시지가 쌓이는 스크롤 가능한 영역
  - 하단 입력 영역 : 메시지 입력창 + 전송 버튼

**각 섹션별 상세**
- 닫힌 상태 버튼 : 60*60px 원형, 브랜드 포인트 컬러를 사용하며 호버 시 약간 커지는 애니메이션 적용
- 열린 상태 채팅창 : 400*500px 크기, 그림자(Shadow) 효과와 둥근 모서리로 화면과 구분되게 처리
- 초기 웰컴 메시지 : 창이 열렸을 때 AI가 먼저 “안녕하세요! 포트폴리오에 대해 궁금한 것이 있으시면 편하게 질문해 주세요.” 메시지 출력
- 로딩(Loading) 상태 : 사용자의 질문 발송 후 AI가 답변을 생성하는 동안 말풍선 안에 '타이핑 애니메이션(점 3개 깜빡임 등)' 표시
- 에러/예외 상태(가드레일) : 포트폴리오와 무관한 질문 시 "저는 포트폴리오에 대한 답변만 가능합니다"와 같은 정중한 우회 안내 메시지 출력

**반응형 동작**
- 모바일 : 버튼 클릭 시 400*500px 창 대신 모바일 화면 전체(전체 화면 모드)를 덮는 UI로 전환되어 타이핑 편의성 확보
- 데스크톱 및 태블릿 : 우측 하단 모서리 부근에 400*500px 사이즈의 패널 형태로 팝업 유지

### 5. 관리자 로그인 화면 (/admin/login)

**레이아웃 구조**
- 화면 배치 : 전체 화면 중앙 정렬(Center Aligned)
- 중앙 로그인 패널 : 서비스 로고 + 이메일/비밀번호 입력 폼 + 로그인 버튼
- 하단 : "홈으로 돌아가기" 링크

**각 섹션별 상세**
- 입력 폼 : 이메일과 비밀번호를 입력할 수 있는 모던하고 깔끔한 형태의 인풋(Input) 필드
- 접근 제어(PRD 반영) : 사전에 허용된 '특정 관리자 이메일(본인 이메일)'로만 접근이 가능하도록 보안 인가 처리
- 에러 피드백 : 권한이 없는 이메일로 시도하거나 비밀번호 오류 시, 폼 하단이나 토스트(Toast) 알림으로 명확한 에러 메시지 노출 ("관리자 권한이 없습니다" 등)

**반응형 동작**
- 공통 : 모바일, 태블릿, 데스크톱 상관없이 로그인 패널의 최대 너비(Max-width: 400px 내외)를 고정하고, 화면 중앙에 위치하도록 여백만 유동적으로 조절

---

### 6. 관리자 대시보드 (/admin)

**레이아웃 구조**
- 좌측 사이드바(LNB) : 메뉴 네비게이션 (프로필 관리, 프로젝트 관리, 챗봇 데이터 관리) 및 로그아웃 버튼
- 우측 메인 영역 : 사이드바에서 선택된 메뉴에 따라 콘텐츠 화면이 전환되는 뷰 영역
- 상단바(모바일 전용) : 모바일 환경을 위한 사이드바 토글(햄버거) 버튼

**각 메뉴별 섹션 상세 (우측 메인 영역)**
- **① 프로필 관리**
  - 구성 : 이름, 한 줄 소개, 프로필 사진 업로드, 스킬 태그, 경력(타임라인)을 수정하는 폼(Form)
  - 액션 : 하단에 "저장하기" 고정 버튼 배치

- **② 프로젝트 관리**
  - 구성 : 상단에 "새 프로젝트 추가" 버튼, 아래에 기존 등록된 프로젝트를 테이블(Table) 형태로 리스트업
  - 액션 : 테이블의 각 항목(Row) 우측에 '수정' 및 '삭제' 아이콘 버튼 제공

- **③ 프로젝트 추가/수정 모달 (Modal)**
  - 구성 : '새 프로젝트 추가' 또는 '수정' 클릭 시 화면 중앙에 나타나는 팝업 창
  - 입력 필드 : 프로젝트 제목, 설명(TextArea/Markdown), 카테고리(Select), 이미지 업로드(File Upload 컴포넌트), 관련 링크(GitHub 등)
  - 액션 : 하단 '취소' 및 '저장' 버튼

- **④ 챗봇 데이터 관리**
  - 구성 : AI 챗봇이 답변 시 참조할 지식 문서나 링크(URL) 데이터 목록 표시
  - 액션 : 텍스트(마크다운) 추가 혹은 링크 추가 폼 제공, 불필요한 참조 데이터 '삭제' 버튼

**반응형 동작**
- 모바일 : 좌측 사이드바는 기본적으로 숨겨져 있으며(Hidden), 상단의 햄버거 메뉴 클릭 시 좌측에서 슬라이드(Drawer) 형태로 나타남. 테이블 영역은 가로 스크롤을 허용하거나 카드(Card) 형태로 레이아웃 변경
- 데스크톱/태블릿 : 좌측 사이드바를 고정(Fixed)하여 항상 노출하고, 우측에 넓은 메인 콘텐츠 영역을 확보
