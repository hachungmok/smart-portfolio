# 기능 및 비기능 요구사항 (Requirements)

> 이 문서는 [PRD_01_Overview.md](./PRD_01_Overview.md) 프로젝트 개요에 이어지는 상세 요구사항 정의서입니다.

## 1. 핵심 기능 (Functional Requirements)

1. **포트폴리오 지식 기반(RAG) 자동 구축 시스템**
   - 개발자가 자신의 이력서(PDF/Markdown)나 GitHub 주소를 입력하면, AI가 이를 분석하여 챗봇이 답변할 수 있는 데이터베이스로 자동 변환합니다.
2. **AI 대화형 포트폴리오 인터페이스 (챗봇)**
   - 방문자가 질문을 하면 포트폴리오 데이터를 바탕으로 자연스럽고 정확하게 답변합니다.
   - **[예외 처리 로직]** 포트폴리오 내용과 무관한 질문(예: 무관한 상식, 날씨, 개인적인 질문 등)을 입력받을 경우, *"죄송합니다. 저는 포트폴리오와 관련된 질문에만 답변할 수 있습니다."* 와 같이 정해진 안내 메시지를 출력하도록 방어 로직을 적용합니다.
3. **방문자 인사이트 대시보드 (관리자 전용)**
   - 방문자들이 챗봇에게 어떤 질문을 가장 많이 던졌는지 통계 및 분석 자료를 대시보드 형태로 제공합니다. 
   - **[접근 제어 로직]** 보안을 위해 로그인 시스템을 도입하며, 오직 사전에 지정된 '특정 이메일'로 로그인한 사용자(포트폴리오 소유자)만이 관리자 페이지에 접근할 수 있도록 권한 검증을 필수로 거칩니다.

## 2. 비기능 요구사항 (Non-Functional Requirements)

- **모바일 최적화 (반응형 웹):** 스마트폰이나 태블릿 등 모바일 기기로 접속하는 방문자가 많을 것을 고려하여, 모든 디바이스에서 챗봇 UI와 포트폴리오 화면이 완벽하게 호환되고 깨지지 않도록 **반응형 웹(Responsive Web)** 디자인을 필수적으로 적용합니다.
- **할루시네이션(환각) 제어:** AI가 포트폴리오에 없는 내용을 지어내서 답변하지 않도록, 오직 '등록된 데이터' 내에서만 답변하게 하는 엄격한 프롬프트(System Prompt)를 구성합니다.
- **개인정보 보호:** 개발자의 연락처나 민감한 정보는 무분별하게 노출되지 않도록 제어합니다.
