# 🚀 프로젝트 개발 마일스톤 및 작업 계획 (PROJECT_PLAN)

PRD, UI 명세, 기술 스택을 종합하여 도출한 전체 개발 마일스톤입니다. 데이터 흐름과 의존성(인증 -> 데이터 입력 -> 데이터 노출 -> AI 학습)을 고려하여 작업 순서를 배치했습니다. 

**[MVP]** : 최소 기능 제품(Minimum Viable Product)을 위해 반드시 구현해야 하는 핵심 기능 (핵심 목표 4가지 반영)
**[확장]** : MVP 릴리즈 이후 고도화 단계에서 구현할 부가 기능

---

## Phase 1 : 프로젝트 기반 설정 (Foundation)
*시스템의 근간이 되는 인프라, 데이터베이스 스키마 및 공통 UI 레이아웃을 세팅합니다.*

- [x] [MVP] Next.js 프로젝트 설정 (완료)
- [ ] [MVP] Supabase 프로젝트 생성 및 데이터베이스 테이블 스키마 구성 (프로젝트, 챗봇 지식용)
- [ ] [MVP] Supabase Storage 버킷 생성 (이미지 업로드용)
- [ ] [MVP] `.env` 환경변수 설정 (Supabase, Gemini API Key 등)
- [ ] [MVP] 디자인 가이드(Tailwind) 설정 및 기본 레이아웃 구성 (GNB, 푸터)

---

## Phase 2 : 관리자 인증 및 데이터 관리 (Admin Backoffice)
*사용자 페이지에서 보여줄 작품(프로젝트)과 데이터를 등록하기 위해 관리자 기능을 먼저 구현합니다.*

- [ ] [MVP] Supabase Auth 기반 관리자 로그인 페이지(`/admin/login`) 구현
- [ ] [MVP] 인증된 사용자만 접근 가능한 라우팅 미들웨어(보안/접근 제어) 적용
- [ ] [MVP] 관리자 대시보드 레이아웃(사이드바) 구성
- [ ] [MVP] 프로젝트(작품) 리스트 조회 및 추가/수정/삭제 폼 구현 (이미지 업로드 포함)
- [ ] [MVP] AI 챗봇이 참조할 지식 데이터(텍스트/URL) 추가 및 관리 기능
- [ ] [확장] 프로필 기본 정보 및 경력(Resume) 데이터 관리 폼 구현

---

## Phase 3 : 사용자 인터페이스 및 데이터 렌더링 (User Interface)
*Phase 2에서 등록한 데이터베이스의 정보들을 실제 방문자가 볼 수 있도록 프론트엔드 화면에 렌더링합니다.*

- [ ] [MVP] 메인 페이지(`/`) 프로필, 스킬, 경력 타임라인 UI 구현
- [ ] [MVP] 프로젝트 목록 페이지(`/project`) 작품 갤러리/그리드 연동 (필터 포함)
- [ ] [MVP] 개별 프로젝트 상세 페이지(`/project/[id]`) 동적 라우팅 및 데이터 연동
- [ ] [확장] 이력 및 경험 상세 화면(`/resume`) 구현 및 PDF 다운로드 기능

---

## Phase 4 : AI 챗봇 및 RAG 연동 (AI Integration)
*완성된 포트폴리오 데이터를 바탕으로 방문자와 소통하는 AI 챗봇을 통합합니다.*

- [ ] [MVP] AI 챗봇 위젯(플로팅 버튼 및 채팅 패널) 공통 UI 컴포넌트 개발
- [ ] [MVP] Next.js API Routes를 이용한 Gemini AI 백엔드 연동
- [ ] [MVP] 데이터베이스의 지식(포트폴리오 내용)을 컨텍스트로 주입하여 답변하는 시스템(RAG) 구현
- [ ] [MVP] 포트폴리오 외의 질문에 방어하는 가드레일 프롬프트 설정
- [ ] [확장] 모바일/태블릿을 위한 AI 챗봇 전용 풀스크린 화면(`/chat`) 구현
- [ ] [확장] 방문자 챗봇 대화 로그 수집 및 대시보드 통계 화면(`/admin/logs`) 연동
