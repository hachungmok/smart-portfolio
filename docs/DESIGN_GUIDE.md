# 디자인 가이드 (DESIGN_GUIDE)

이 문서는 Smart Portfolio 웹사이트의 전반적인 디자인 톤 앤 매너(Tone & Manner), 색상, 타이포그래피, 그리고 UI 요소들의 스타일링 기준을 정의합니다.

## 🎯 디자인 컨셉 (Design Concept)
- **핵심 키워드**: 깔끔함(Clean), 미니멀리즘(Minimal), 전문성(Professional), 부드러움/따뜻함(Warmth)
- **기본 테마**: 다크 모드 (Dark Mode Default)
- **방향성**: 차갑고 딱딱한 IT 포트폴리오가 아닌, 신뢰감을 주면서도 인간적인 온기가 느껴지는 세련된 다크 테마

---

## 🎨 주요 색상 팔레트 (Color Palette)

다크 모드의 차가운 느낌을 중화시키기 위해, 배경은 무채색보다는 약간의 웜톤(Warm tone)이 섞인 다크 그레이를 사용하고, 포인트 컬러로 따뜻한 베이지/앰버 계열을 적용합니다.

| 용도 | 색상 이름 | HEX 코드 | 설명 |
| :--- | :--- | :--- | :--- |
| **Background** (기본) | Warm Dark | `#18181B` | 화면 전체 배경. 완전한 검은색이 아닌 부드러운 다크 징크(Zinc) |
| **Surface** (카드/패널) | Surface Gray | `#27272A` | 카드, 모달, 네비게이션 바 등의 표면 배경색 |
| **Primary** (브랜드/주요) | Soft Amber | `#E5A970` | 주요 버튼(CTA), 강조 텍스트, 활성 탭에 사용되는 따뜻한 베이지-오렌지 |
| **Secondary** (보조) | Muted Gray | `#3F3F46` | 보조 버튼, 비활성 아이콘, 구분선(Border) 색상 |
| **Accent** (포인트/알림) | Soft Coral | `#E07A5F` | 경고, 알림 뱃지, 호버 포인트 등 시선을 끌어야 할 때 사용 |
| **Text** (기본 텍스트) | Off White | `#F4F4F5` | 제목 및 본문의 기본 텍스트 (눈부심 방지를 위해 순백색 지양) |
| **Text Muted** (보조 텍스트)| Dimmed Gray | `#A1A1AA` | 부가 설명, 날짜, 플레이스홀더 등에 사용되는 텍스트 |

## ✍️ 타이포그래피 (Typography)

깔끔하고 현대적인 가독성을 제공하면서도, 너무 차갑지 않은 서체를 조합합니다.

1. **제목용 폰트 (Headings)**
   - **영문**: `Outfit` (둥글면서도 기하학적인 모던함 제공)
   - **한글**: `Pretendard` (프리텐다드 - 압도적인 가독성과 중립성)
   - **적용**: 로고 텍스트, 화면 대제목(h1), 섹션 제목(h2), 프로젝트 타이틀 등

2. **본문용 폰트 (Body)**
   - **영문 & 한글**: `Pretendard`
   - **적용**: 본문 텍스트, 설명, 버튼 라벨, 태그 등
   - **특징**: 자간(Letter-spacing)을 `-0.02em` 정도로 살짝 좁게 설정하여 밀도 있고 정돈된 느낌 부여. 본문 줄 간격(Line-height)은 `1.6` 유지.

---

## 📐 여백 및 간격 기준 (Spacing System)

8pt Grid System을 기반으로 하여 일관된 여백을 제공합니다.

- **미세 간격 (Micro)** : `4px`, `8px` (아이콘과 텍스트 사이, 태그 내부 여백)
- **요소 간격 (Item)** : `16px`, `24px` (리스트 아이템 사이, 문단 사이, 버튼 간격)
- **섹션 내부 여백 (Component)** : `32px`, `48px` (카드 내부 패딩, 섹션 제목과 본문 사이)
- **섹션 간 여백 (Section)** : `80px` ~ `120px` (히어로 영역과 프로젝트 영역 사이 등 큼직한 구분)

---

## 🔲 UI 요소 스타일링 (UI Components)

미니멀리즘을 유지하면서 부드러움을 더하기 위해 모서리를 적절히 둥글게 처리합니다.

**1. 모서리 둥글기 (Border Radius)**
- **입력 필드 (Input) & 작은 버튼** : `8px` (너무 둥글지 않은 단정한 형태)
- **카드 컴포넌트 (프로젝트 카드 등)** : `16px` (시각적으로 안정감을 주는 부드러운 형태)
- **태그/뱃지** : `999px` (완전한 알약 형태, 둥글게)
- **플로팅 챗봇 버튼 / 프로필 사진** : `50%` (완벽한 원형)

**2. 그림자 스타일 (Shadows)**
다크 모드에서는 그림자가 잘 보이지 않으므로, 넓게 퍼지는 어두운 그림자와 함께 윗면(Top)에 미세한 밝은 테두리를 주어 깊이감을 만듭니다.
- **기본 카드 (Resting)**
  - Box Shadow: `0 4px 20px rgba(0, 0, 0, 0.4)`
  - Border: `1px solid rgba(255, 255, 255, 0.05)` (은은한 외곽선으로 카드 형태 구분)
- **호버 상태 카드 (Hover / Active)**
  - Box Shadow: `0 8px 30px rgba(0, 0, 0, 0.6)`
  - Transform: `translateY(-4px)` (살짝 위로 떠오르는 효과)
  - 마우스를 올리면 카드가 떠오르면서 그림자가 더 짙어지고 넓게 퍼지는 입체감 제공.

**3. 버튼 호버 효과 (Button Transitions)**
- 배경색이 살짝 밝아지거나 투명도가 조절되며, 모든 변화는 `transition: all 0.2s ease-in-out;` 을 적용하여 부드럽고 쫀득한 느낌을 줍니다.

## 📱 반응형 디자인 기준 (Responsive Breakpoints)

다양한 디바이스 환경에서 일관된 사용자 경험을 제공하기 위해 다음의 3단계 브레이크포인트(Breakpoint)를 기준으로 레이아웃이 변환됩니다.

1. **모바일 (Mobile) : `0px` ~ `768px`**
   - **레이아웃**: 1열(Column) 세로 배치 레이아웃
   - **네비게이션**: 상단 헤더에 햄버거 메뉴 적용 (클릭 시 전체 화면 오버레이)
   - **위젯/모달**: AI 챗봇 창 열림 시 모바일 화면을 꽉 채우는 전체 화면 모드로 전환하여 타이핑 공간 확보

2. **태블릿 (Tablet) : `769px` ~ `1024px`**
   - **레이아웃**: 프로젝트 카드 등 2열(Column) 그리드 배치
   - **네비게이션/사이드바**: 관리자 대시보드의 경우 아이콘만 보이는 축소된(Collapsed) 사이드바로 변환되어 메인 콘텐츠 공간 확보

3. **데스크톱 (Desktop) : `1025px` 이상**
   - **레이아웃**: 3열(Column) 그리드 배치 등 가장 넓고 쾌적한 뷰 제공
   - **네비게이션/사이드바**: 풀 네비게이션 메뉴 노출 및 관리자 페이지 좌측 텍스트 포함 풀 사이즈 사이드바 고정
